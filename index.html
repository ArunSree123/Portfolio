AI-Based Fraud Detection System

Real-time transaction analysis · Final Year
ML · Final Year
Problem
Fraudulent digital transactions are hard to catch with static rule-based systems because fraud patterns shift constantly. A model needs to learn from historical data.
Approach
Built an ML pipeline covering data preprocessing, feature engineering, model training using classification algorithms, and evaluation — with extra care around class imbalance.
Outcome / Learning
Learned that accuracy is misleading with imbalanced datasets. Shifted focus to precision, recall, and F1 score. Used SMOTE for oversampling minority (fraud) samples.
Key Challenge
Managing extreme class imbalance — real fraud cases are rare by definition — and selecting the right evaluation metric for a skewed binary classification problem.
Python scikit-learn Pandas NumPy Matplotlib SMOTE
